# Templates 使用指南

本目录包含不同类型笔记的模板文件，用于标准化知识管理。

## 📚 可用模板

### 1. 论文笔记模板 (`paper-template.md`)

**适用于**: 学术论文、研究报告

**使用方法**:
```bash
cp templates/paper-template.md LLM/新论文名称.md
```

**特点**:
- 结构化的论文信息记录
- 核心贡献和技术细节分析
- 个人思考和应用场景
- TODO跟踪

### 2. 技术文章模板 (`blog-template.md`)

**适用于**: 技术博客、教程文章、技术文档

**使用方法**:
```bash
cp templates/blog-template.md 编程语言/新主题.md
```

**特点**:
- TL;DR快速总结
- 代码示例和命令
- 实践记录
- 后续行动项

### 3. 项目笔记模板 (`project-template.md`)

**适用于**: 个人项目、实践项目、技术实现

**使用方法**:
```bash
cp templates/project-template.md 数据库/项目名.md
```

**特点**:
- 完整的项目文档结构
- 架构设计和实现细节
- 性能数据和效果记录
- 技术难点和解决方案
- 学习收获总结

## 💡 使用建议

1. **复制而非移动**: 使用 `cp` 命令复制模板，保持模板文件完整
2. **根据需要调整**: 模板是起点，可以根据实际需求增删章节
3. **填写标签**: 记得在文章末尾添加相关标签，便于后续检索
4. **更新状态**: 使用模板末尾的状态标记追踪学习进度
5. **交叉引用**: 在"相关笔记"部分添加链接，构建知识网络

## 🔄 模板更新

如果你发现某种笔记类型经常使用，可以创建新的模板文件：
1. 在 `templates/` 目录创建新的 `.md` 文件
2. 设计适合该类型的结构
3. 更新本 README.md 文档

## 📝 示例

### 使用论文模板的示例流程

1. 复制模板:
   ```bash
   cp templates/paper-template.md LLM/PromptBreeder论文笔记.md
   ```

2. 填写基本信息和核心内容

3. 阅读过程中记录关键要点

4. 完成后添加个人思考和TODO

5. 在Timeline中引用该笔记

### Timeline条目示例

```markdown
## 2025-01-07 新论文阅读

* PromptBreeder论文深度解读
  * 核心思想: 自动化提示词优化
  * 详见: [PromptBreeder论文笔记](../LLM/PromptBreeder论文笔记.md)
  * TODO: 实现简化版本测试效果
```
